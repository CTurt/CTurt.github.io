FreeBSD kernel bugs

I'm currently a member of the HardenedBSD team, where my role is to identify, analyse, and report vulnerabilities and other general improvements to the FreeBSD kernel. I initially submit patches to HardenedBSD which will later be considered for upstreaming to FreeBSD.

Below is a list of my bug reports, as well as the link to the accepted patch in FreeBSD, where applicable; most of these are not critical since they are only triggerable as root, or have other, similar limitations:

  • witness_initialize() does not perform bound checking of witness_count
  • Improper userland pointer handling in aacraid
  • hpt_set_info buffer overflow
  • Fix double strlen in ktrstruct - Accepted patch
  • Use of uninitialised stack data in bxe device
  • Integer overflow in nfssvc system call
  • OGIO_KEYMAP command does not restore priority level
  • kbd race attacks
  • Out of bounds negative array index in iicrdwr - Accepted patch
  • Use of initialised stack variables in tdfx_query_update
  • Kernel stack overflow in sysctl handler for kern.binmisc.add
  • Iconv uses strlen directly on user supplied memory
  • Memory leak in ctl.c
  • Negative array index in ctl.c
  • Integer overflow in sysctl_kern_proc_args
  • kiconv reference count integer overflow
  • IPFW firewall heap overflow
  • Check for negative lengths in m_pulldown
